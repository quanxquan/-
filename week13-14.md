## 一、前端与伺服器沟通的三种方式

| 方式 | 特色 | 适用场景 |
|------|------|----------|
| **表单（Form）** | 每次送出都换页，主导权在伺服器 | 简单的登入/注册/提交 |
| **Fetch API** | 不换页，主导权在 JavaScript | 单页应用（SPA）、动态更新内容 |
| **WebSocket** | 双向即时连线，更流畅 | 即时通讯、多人协作（下週介绍） |

## 二、Fetch API 基本概念

### 什么是 Fetch API？

Fetch API 是现代浏览器内建的工具，让 JavaScript 可以在**不换页的情况下**向伺服器请求资料，并动态更新页面内容。

### 基本写法（推荐用 async/await）

```javascript
async function getData() {
    const response = await fetch('/api/posts');  // 向伺服器请求资料
    const data = await response.json();          // 将回应转成 JSON 物件
    console.log(data);
}
```

### 与旧写法（`.then()` 回呼）的比较

```javascript
// 旧写法（回呼地狱）
fetch('/api/posts')
    .then(response => response.json())
    .then(data => {
        // 处理资料
        console.log(data);
    });

// 新写法（async/await，推荐）
const response = await fetch('/api/posts');
const data = await response.json();
console.log(data);
```

> ✅ `await` 写法读起来像正常的程序，逻辑清晰，不会造成多层巢状（三角化问题）。

## 三、单页应用（SPA）

### 什么是 SPA？

Single Page Application，整个网站只有**一个 HTML 页面**，画面切换由 JavaScript 控制，不重新载入页面。

### URL 的 Hash（书签）机制

SPA 用 `#` 符号后面的内容来区分「目前在看哪一页」：

```
http://localhost:8000/index.html       ← 列出所有贴文
http://localhost:8000/index.html#post0 ← 显示第 0 篇贴文
http://localhost:8000/index.html#new   ← 新增贴文页面
```

URL 变了，但**页面不会重新载入**，只有 `#` 后面的内容改变。

### 监听 Hash 变化

```javascript
window.onhashchange = function() {
    const hash = location.hash;  // 取得 # 后面的内容
    
    if (hash === '#new') {
        showNewPostForm();
    } else if (hash.startsWith('#post')) {
        const id = hash.replace('#post', '');
        showPost(id);
    } else {
        showList();  // 预设显示列表
    }
};
```

## 四、SPA 架构下的前后端分工

### 后端（Python/FastAPI）

只负责**回传 JSON 资料**，不再回传 HTML：

```python
@app.get("/posts")
def list_posts():
    return posts  # 回传 JSON，不是 HTML

@app.get("/posts/{id}")
def get_post(id: int):
    return posts[id]
```

### 前端（JavaScript）

负责**接收 JSON、自己产生 HTML**，塞入页面：

```javascript
async function showPost(id) {
    const response = await fetch(`/posts/${id}`);
    const post = await response.json();
    
    // 自己产生 HTML 塞入页面
    document.getElementById('content').innerHTML = `
        <h1>${post.title}</h1>
        <p>${post.body}</p>
    `;
}
```

### 表单 vs Fetch API 对比

| | 表单方式 | Fetch API |
|--|---------|-----------|
| 主导权 | 伺服器 | JavaScript（前端） |
| 是否换页 | ✅ 会换页 | ❌ 不换页 |
| 使用模板引擎 | ✅ 需要（Jinja2） | ❌ 不需要 |
| 后端回传格式 | HTML | JSON |
| SEO 友善度 | ✅ 较好 | ⚠️ 较差（需额外处理） |

## 五、SPA 的优缺点

### 优点

- 流量较小（只传 JSON，不传整个 HTML）
- 体验流畅，不会白画面
- 可实现复杂的单页互动（如 Google Maps，无法用表单实现）

### 缺点（SEO 问题）

- Google 爬虫爬到的 HTML 几乎是空的，**内容都由 JavaScript 动态产生**
- 搜寻引擎可能无法正确索引页面内容
- Google 需要用「无头浏览器（Headless Browser）」才能爬到内容，对 Google 来说负担较重

> 💡 若网站需要被搜寻引擎找到（如电商、部落格），需额外做 **SSR（Server-Side Rendering）** 处理。

## 六、async/await 与非阻塞（Non-blocking）概念

### 为什么要用 async/await？

伺服器同时服务多个使用者，如果一个请求在等待 I/O（如读档、存取资料库），不能卡住其他人的请求。

```
使用者 A 请求 → 等待资料库查询（I/O）
                ↓ 暂时让出控制权
使用者 B 请求 → 执行中...
使用者 C 请求 → 执行中...
                ↓ 资料库回应了
使用者 A 请求 → 继续执行，回传结果
```

### Blocking vs Non-blocking

| 模式 | 说明 | 优缺点 |
|------|------|--------|
| **Blocking**（阻塞） | 等 I/O 完成才继续 | 好写，但效率差 |
| **Non-blocking**（非阻塞） | I/O 进行中，先做别的事 | 效率好，较难写 |

> `async/await` 是 Non-blocking 的语法糖，让程式看起来像 Blocking（线性），但实际上是 Non-blocking，**两全其美**。

## 七、解决浏览器快取问题

开发时若发现画面没有更新（快取住），可改用不同的 Port：

```bash
fastapi dev --port 8001  # 换个 Port，就不会有快取问题
```

## 九、Ollama 本地 AI 整合

### 什么是 Ollama？

Ollama 是一个**在本机自架 AI 模型**的软体，让你不需要依赖 OpenAI API，就能在自己的电脑上运行 AI 模型。

- **Ollama** = 软体（管理与执行模型的工具）
- **LLaMA** = 模型（Meta/Facebook 开源的语言模型）

### 常见可用模型

| 模型 | 来源 | 建议配备 |
|------|------|---------|
| LLaMA 3.2 3B | Meta | 4GB+ RAM |
| Gemma | Google | 4GB+ RAM |
| Qwen | 阿里巴巴 | 4GB+ RAM |
| DeepSeek | DeepSeek | 视版本而定 |
| Phi | Microsoft | 较小，适合低配 |

> ⚠️ 建议只装 7B 以下的模型，8GB RAM 约只能跑 4B\~7B，27B 需要约 27GB RAM。

### 使用方式

Ollama 启动后会在 `localhost:11434` 提供 API：

```python
import requests

response = requests.post('http://localhost:11434/api/generate', json={
    'model': 'llama3.2',
    'prompt': '请介绍牛顿三大定律',
    'stream': False
})

print(response.json()['response'])
```

### 整合进 FastAPI

```python
@app.post("/chat")
async def chat(question: str):
    response = requests.post('http://localhost:11434/api/generate', json={
        'model': 'llama3.2',
        'prompt': question,
        'stream': False
    })
    return {"answer": response.json()['response']}
```
